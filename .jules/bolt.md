## 2024-06-18 - Missing composite index for frequent Task queries
**Learning:** The application frequently queries tasks filtered by `userId`, `deletedAt`, and optionally `status` (e.g., in `packages/web/src/server/tasks.ts` and `packages/web/src/app/api/tasks/route.ts`). While single-column indexes existed on `userId` and `deletedAt`, a composite index was missing. This is a critical architectural performance bottleneck as the tenant-isolated data grows, as the database engine would still need to filter many rows after utilizing a single index.
**Action:** Always add composite indexes aligned with the specific filtering patterns of the queries. In this case, adding `@@index([userId, deletedAt, status])` significantly speeds up the most common query paths for fetching a user's tasks.
